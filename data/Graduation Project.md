## **Group Project Title (Shared)**

**Developing a Multi-Agent RAG-Based System to Improve Automation in the Software Development Lifecycle**

### **1\. Shared System Architecture**

* Multi-agent design (e.g. Analyst, Developer, QA, DevOps agents)

* Agent coordination and task delegation

* RAG pipeline (ingestion → retrieval → generation)

### **2\. Shared Knowledge Base**

* SDLC artefacts:

  * requirements documents

  * design guidelines

  * coding standards

  * testing strategies

  * CI/CD and deployment practices

* Knowledge stored in a **domain-agnostic format**

* Designed for **reuse and extension**

### **3\. Core Use Cases (Group-Level)**

* Querying SDLC knowledge

* Supporting decision-making (e.g. design trade-offs, testing strategies)

* Demonstrating end-to-end SDLC support

### **4\. Shared Evaluation**

* Functional correctness

* Knowledge retrieval quality

* System usability (qualitative evaluation)

# **INDIVIDUAL GRADUATION PROJECT** 

After the group project is completed, **each student writes an individual graduation report**, using the shared system as a **baseline platform**.

You will:

* **reuse the group-built knowledge base and architecture**

* **extend or specialise it** with your own research focus

Examples of **individual differentiation**:

### **Option 1 – Agent Reasoning & Coordination**

* Improve planning, memory, or task decomposition of agents

* Compare single-agent vs multi-agent decision quality

### **Option 2 – Knowledge Representation & Retrieval**

* Improve chunking, metadata, or retrieval strategy

* Evaluate impact on decision accuracy

### **Option 3 – SDLC Automation Focus**

* Deep dive into one SDLC phase:

  * requirements analysis

  * testing strategy

  * DevOps / deployment decisions

### **Option 4 – Generalisation Study**

* Apply the same architecture to:

  * IT operations knowledge

  * business process documentation

* Show minimal but meaningful generalisation

# **Summary**

The group project focuses on developing a shared Multi-Agent RAG-based SDLC knowledge system, including architecture design, knowledge base construction, and a working prototype. Each member will then reuse this shared system as a baseline and conduct an individual graduation project by extending or specialising one aspect of the system, such as agent reasoning, knowledge retrieval, SDLC phase automation, or domain generalisation, ensuring clear individual contributions.
